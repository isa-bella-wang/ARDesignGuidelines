Foundation

Why Limit Information?

AR interfaces operate within a see-through environment where users must process both digital overlays and the real-world scene simultaneously. This makes working memory management central: users can maintain only a small number of items in conscious focus at once, and each additional element competes with real-world stimuli.

Impact on AR Design

High information density increases visualization time disproportionately, causes increased cognitive load, delays visual search, makes tasks feel harder, and impacts usability as a whole. In AR, this is compounded by the need to also process real-world stimuli.

Design Principle for AR Information Density

The DOI framework provides a mathematical approach to managing information density in AR. By combining intrinsic importance (API) with contextual relevance (distance penalty), designers can create interfaces that automatically adapt to user focus and context. Set API ratings for all objects based on task relevance, calculate distance penalties based on physical proximity and viewing angle, compute DOI scores, and apply display thresholds to show full cards (DOI 9+), labels (DOI 6-8), clusters (DOI 3-5), or hide content (DOI 0-2). This approach maintains cognitive load within working memory limits while preserving awareness of available information.
